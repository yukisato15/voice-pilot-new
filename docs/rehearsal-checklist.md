# Zoom 共有リハーサル手順（SharedStage 終了演出／スライド読み込み）

## 目的
- SharedStage の終了10秒カウントダウン〜STOP 演出が Zoom 画面共有経由でも視認性・パフォーマンスともに十分か確認する。
- ガイダンススライド（画像フォルダ読み込み機能）が実機で問題なく動作するかを検証し、スライド解像度や切り替え遅延の有無を把握する。

## 前提準備
1. `npm run dev --prefix app` でアプリを起動し、SharedStage を Zoom 側へ共有できる状態にする。
2. PowerPoint から書き出した PNG/JPEG 画像を `slides rehearsal/` 等のフォルダにまとめておく（ファイル名は `slide_01.png` のように昇順）。
3. Zoom 会議を立ち上げ、録画を行わなくても良いが、録画ボタン UI が表示される環境で確認する。

## 手順
1. **スライド読み込み**
   - DirectorConsole の「PPT/画像フォルダを読み込む…」から準備したフォルダを指定。
   - SharedStage でスライドが即座に描画されるか確認。
   - 前後ボタンでの切り替え時に遅延や描画崩れがないかチェック。
   - Zoom 画面共有で 720p/1080p どちらの場合でも文字が潰れないかを確認（可能であれば双方）。
2. **終了演出の確認**
   - タイマーを開始し、SharedStage での表示を Zoom で共有。
   - カウントダウン残り 60 秒で黄色メッセージ、10 秒で赤オーバーレイが表示されるか視覚的に確認。
   - STOP 表示が 0 秒で切り替わり、数秒間継続するか確認。
   - Zoom 共有側で FPS が落ちたり、表示が固まる現象が無いか観察。
3. **チェックリスト**
   - [ ] スライド読み込み後すぐに SharedStage に反映された。
   - [ ] スライド切り替え時に 0.5 秒以上の遅延が発生しない。
   - [ ] 60 秒メッセージが黄色トーンで視認できる。
   - [ ] 10 秒カウントが 1 秒間隔で減少し、全画面赤オーバーレイが発生した。
   - [ ] STOP 表示でテキストが潰れず、注意喚起として十分なサイズである。
   - [ ] Zoom 側に録画される映像でも STOP が明瞭に映っている（任意で録画を再生確認）。

## 確認後のメモ
- 気付いた点（色味の調整、フォントサイズ、スライド比率など）を `docs/ui-review-notes.md` に追記するか、GitHub Issue 化する。
- パフォーマンス問題があれば `SharedStage` の画像ロード戦略やオーバーレイの処理を検討。
